# Skill — QA audit

Goal: verify that the application works and meets acceptance criteria.

Checklist:
- Docker Compose builds.
- Backend `/health` works.
- API docs open.
- Database migrations run.
- Seed data loads.
- Frontend opens.
- Map renders.
- Dashboard renders.
- Mechanisms list.
- Project/investment creation works.
- Prioritization ranking works.
- MRV observation can be created.
- Data gaps are visible.
- Tests run.
- No secrets committed.
- No destructive scripts included.

Output:
- `docs/QA_REPORT.md`
- list of passed checks;
- list of failed checks;
- fixes applied;
- remaining issues.
